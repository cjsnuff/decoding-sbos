Error type and cause
- Type: runtime/configuration error during config load.
- Cause: vite.config.ts requires a PORT environment variable and throws when it is not defined. Netlify's build environment does not provide PORT by default, so the config throws during build time and stops the build.

Solution (two options — prefer option A)
A — Recommended: make vite.config.ts tolerant of missing PORT during build
- Adjust vite.config.ts so it only requires PORT when running the dev server (vite serve) or provide a sensible default. This prevents the build from failing on Netlify where a dev server port is irrelevant.

Example change (replace the strict throw with a safe default or only require for serve):
```ts
// vite.config.ts
import { defineConfig } from 'vite';

export default defineConfig(({ command }) => {
  if (command === 'serve') {
    // for local dev: prefer environment PORT but fall back to 5173
    const port = process.env.PORT ? Number(process.env.PORT) : 5173;
    return {
      server: { port },
      // other dev-only config...
    };
  }

  // build: do not require PORT
  return {
    // build-specific config here
  };
});
```
- Commit this change and push; Netlify builds will then complete without needing a PORT env var.

B — Alternate: set PORT in Netlify build environment (if you prefer keeping the strict requirement)
- In the Netlify UI: Site > Site settings > Build & deploy > Environment > Environment variables > Add variable
  - Key: PORT
  - Value: 5173 (or whichever port your config expects)
- Or add it in netlify.toml under [build.environment]:
```toml
[build.environment]
PORT = "5173"
```
- Verify the variable is added and re-trigger a deploy.
